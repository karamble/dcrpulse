# Decred Pulse for TrueNAS

This directory contains the **TrueNAS SCALE app** packaging for Decred Pulse,
ready to be submitted to the [TrueNAS community apps catalog](https://github.com/truenas/apps)
(the `community` train).

It deploys the full Decred Pulse stack as seven containers behind a single web
dashboard:

| Container | Role |
|-----------|------|
| `dcrd` | Decred full node (blockchain sync, P2P, RPC) |
| `dcrwallet` | HD wallet with staking / mixing support |
| `dcrlnd` | Lightning Network node |
| `brclientd` | Bison Relay client (encrypted messaging) |
| `dcrdex` | Decentralized exchange client |
| `dashboard` | Web UI / block explorer (the portal) |
| `tor` | Optional Tor SOCKS proxy for the stack |

All images are pulled from GitHub Container Registry (`ghcr.io/karamble/dcrpulse-*`);
nothing is built on the TrueNAS host.

## Layout

This is the native TrueNAS app format (not a runnable `docker-compose.yml` like the
CasaOS / Umbrel packaging). The compose file is a Jinja2 template that the TrueNAS
renderer turns into a real compose file using the values the user picks in the UI.

```
dcrpulse-truenas/
├── app.yaml                       # App metadata (versions, maintainer, icon, ...)
├── item.yaml                      # Catalog entry (category, tags, icon)
├── ix_values.yaml                 # Static values: container images + constants
├── questions.yaml                 # The configuration form shown in the TrueNAS UI
├── README.md                      # This file
└── templates/
    ├── docker-compose.yaml        # Jinja2 template (uses the ix-lib render API)
    └── test_values/
        └── basic-values.yaml      # CI test scenario
```

> Note: `templates/library/` (the rendering library), `lib_version_hash` and
> `item.yaml` are normally (re)generated by the TrueNAS `ci.py` tooling at submission
> time, so they are intentionally **not** vendored here.

## Configuration

The TrueNAS UI exposes:

- **RPC username / password** – shared by `dcrd`, `dcrwallet` and `dcrdex`. Change the
  password before exposing the node.
- **Wallet address gap limit** – default `400`.
- **Transaction index** (`--txindex`) – on by default, required for the block explorer.
- **Lightning testnet** toggle.
- **Web UI port** (default `8080`) and **dcrd P2P port** (default `9108`).
- **Storage** – six datasets (blockchain+wallet data, dcrlnd, brclientd, dcrdex,
  dashboard, tor), each selectable as an automatically created ixVolume or an existing
  host path.
- Standard **labels** and **resource limits**.

## Security / runtime notes

- All containers run as a non-root user/group (`1000:1000`); a permissions init
  container fixes dataset ownership on first start.
- The shared `data` dataset is mounted read-write into `dcrd`/`dcrwallet`/`dashboard`
  and read-only into the other services so the dcrd RPC certificate is shared safely.
- Only the dashboard web port and the dcrd P2P port are published; all inter-service
  RPC stays on the app's internal Docker network.
- Initial blockchain sync can take several hours and needs ~15GB+ of disk space.

## Testing locally

With Docker available, from a checkout of the
[truenas/apps](https://github.com/truenas/apps) repo (copy this directory to
`ix-dev/community/dcrpulse/` first):

```bash
# Render only (no deploy)
./.github/scripts/ci.py --app dcrpulse --train community \
  --test-file basic-values.yaml --render-only=true

# Full deploy + health monitoring
./.github/scripts/ci.py --app dcrpulse --train community \
  --test-file basic-values.yaml --wait=true
```

## Submitting to the TrueNAS catalog

1. Fork [truenas/apps](https://github.com/truenas/apps).
2. Copy this directory to `ix-dev/community/dcrpulse/`.
3. Run `ci.py` (above) to generate `templates/library/`, `item.yaml` and
   `lib_version_hash`, and to validate the render.
4. Open a PR against `truenas/apps` following its
   [CONTRIBUTIONS.md](https://github.com/truenas/apps/blob/master/CONTRIBUTIONS.md).
   The icon and screenshots in [`../dcrpulse-casaos/`](../dcrpulse-casaos/) can be
   provided to the reviewer for upload to the TrueNAS CDN.
